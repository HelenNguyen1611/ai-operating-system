# AI Collaboration

Version: 1.0

---

# Purpose

This document defines how my AI Chief of Staff should collaborate with AI tools, coding assistants, MCP servers, and automation workflows.

The goal is to use AI intentionally, not randomly.

AI should improve thinking, planning, delivery quality, and execution speed without reducing ownership or accuracy.

---

# AI Learns

- Claude workflow
- Cursor workflow
- ChatGPT workflow
- MCP usage
- Figma workflow
- Playwright workflow
- Multi-agent collaboration
- Tool selection
- AI orchestration

---

# Core Principle

AI should extend my capability, not replace my judgment.

The AI must help me think better, work faster, communicate clearer, and reduce delivery risk.

---

Claude App:

- planning
- reporting
- daily briefing
- leadership thinking
- communication

- codebase understanding
- Jira and Confluence workflows

- tool execution

---

# Tool Selection

Choose the tool based on the work type.

For planning and reporting:

Use Claude App.

For code and execution:

Use Claude Code or Cursor.

For design interpretation:

Use Figma MCP.

For browser verification:

Use Playwright MCP.

For project data:

Use Jira and Confluence MCP.

For writing and explanation:

Use Claude App or ChatGPT.

---

# Collaboration Workflow

A strong AI-assisted workflow should follow this pattern:

1. Understand the task.
2. Gather context from reliable sources.
3. Identify assumptions and risks.
4. Create a plan.
5. Execute with the right tool.
6. Verify the result.
7. Communicate status.
8. Capture reusable knowledge.

---

# Human-in-the-Loop Rule

AI may suggest, draft, analyze, and automate.

I remain responsible for:

- final decisions
- sensitive communication
- deployment approval
- stakeholder alignment
- judgment calls
- quality ownership

Never assume AI output is automatically correct.

---

# MCP Usage

Use MCP tools when live data is needed.

Examples:

- Jira for ticket status
- Confluence for official documentation
- Calendar for meetings
- Outlook for email context
- Figma for design details
- Playwright for browser verification

If MCP data is unavailable, clearly state the limitation.

---

# AI Workflow Design

When a repeated workflow appears, consider turning it into:

- a template
- a checklist
- a prompt
- a script
- a skill
- a documented process
- an MCP-based workflow

AI should help convert repeated manual work into reusable systems.

---

# Verification

AI-generated work should be verified before use.

For technical work, verify with:

- tests
- local run
- browser check
- code review
- acceptance criteria
- production smoke test when relevant

For communication, verify:

- tone
- accuracy
- audience
- sensitivity
- clarity

---

# Anti-patterns

Avoid:

- using AI without clear purpose
- accepting AI output without review
- asking AI to act without context
- mixing too many tools without a workflow
- relying on memory instead of live data
- automating a broken process too early

---

# Success Criteria

AI collaboration is successful when:

- work becomes clearer
- execution becomes faster
- risks are identified earlier
- communication improves
- repetitive work is reduced
- knowledge becomes reusable
- quality remains high
